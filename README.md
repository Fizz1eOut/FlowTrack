# FlowTrack
 
**FlowTrack** — это современное прогрессивное веб-приложение (SPA) для управления задачами и командными рабочими пространствами. Пользователь может зарегистрироваться, настроить профиль (имя, аватар, часовой пояс, рабочие часы), создавать личные и командные воркспейсы, управлять задачами в нескольких режимах (список, Kanban, архив), отслеживать время через встроенный таймер, приглашать участников команды, получать уведомления и отслеживать прогресс через систему XP и уровней..
 
---
 
# Основные возможности
 
## Регистрация и вход
 
* **Регистрация** по email и паролю с клиентской валидацией; дублирование email распознаётся и сопровождается понятным сообщением.
* **Вход** с проверкой учётных данных и отображением ошибок.
* **Восстановление пароля:** отправка письма со ссылкой для сброса и отдельная страница установки нового пароля.
* **Смена пароля** через профиль.
* **Защита от двойной отправки:** блокировка повторного запроса регистрации до завершения предыдущего.
* **Обработка ожидающих приглашений:** если пользователь переходил по ссылке-инвайту до авторизации — после входа он автоматически попадает на страницу принятия приглашения.
 
## Онбординг нового пользователя
 
* После регистрации запускается пошаговый онбординг: выбор типа воркспейса (личный или командный).
* Настройка рабочих часов и часового пояса для персонализированного отображения задач.
* Проверка наличия воркспейса при каждом входе — пользователь без воркспейса перенаправляется на онбординг автоматически.
 
## Профиль пользователя
 
### Просмотр информации
 
Отображаются имя (**full_name**), email, дата регистрации (форматирована на русском) и аватар с автоматически вычисляемыми инициалами как резервный вариант.
 
### Редактирование имени
 
* Клиентская валидация имени и немедленное обновление кэша после сохранения.
* Отображение сообщения об успешном обновлении.
 
### Аватар
 
* Загрузка файла изображения → загрузка в Supabase Storage → обновление публичного URL в профиле → немедленное обновление кэша для UI.
* Автоматическое вычисление инициалов (одно или два слова в имени) для резервного отображения.
* Кэширование профилей на клиенте по `userId` для снижения числа запросов и ускорения рендера.
 
### Смена пароля
 
* Смена пароля через Supabase Auth с отображением результата.
 
## Рабочие пространства (Workspace)
 
* Поддержка двух типов воркспейсов: **личный** (Personal) и **командный** (Team) — пользователь может иметь один личный и неограниченное количество командных.
* Создание, редактирование и удаление воркспейсов (личный воркспейс защищён от удаления).
* Переключение текущего воркспейса с сохранением выбора в `localStorage` — при перезагрузке страницы активный воркспейс восстанавливается.
* Автоматический выбор воркспейса при первом входе: приоритет отдаётся личному, иначе — первому в списке.
* Задачи, участники и приглашения всегда привязаны к конкретному воркспейсу.
 
## Управление участниками воркспейса
 
* Приглашение участников по email с указанием роли.
* Просмотр списка текущих участников и их ролей.
* Смена роли участника (например, с member на admin).
* Удаление участника из воркспейса.
* Просмотр активных приглашений и отмена ещё не принятых.
* Фильтрация приглашений по статусу: не принятые и не просроченные.
* **Принятие приглашения по токену:** переход по ссылке-инвайту из письма принимает приглашение и добавляет пользователя в воркспейс.
 
## Задачи
 
### Основные операции
 
* Создание задач с привязкой к воркспейсу и пользователю; автоматическая нумерация задач через DB-триггер.
* Редактирование полей задачи: название, описание, статус, дедлайн, приоритет, исполнитель.
* Удаление задачи.
* Назначение задачи на конкретного пользователя воркспейса (с учётом ролей).
 
### Статусы и Kanban
 
* Задачи поддерживают статусы: `backlog`, `in_progress`, `done`, `archived`.
* Kanban-доска с группировкой задач по статусам, drag-and-drop перемещением карточек и обновлением статуса через `updateTaskStatus`.
* Мгновенная оптимистичная UI-реакция: статус меняется в локальном стейте до завершения запроса; при ошибке — автоматический откат.
 
### Страница Tasks и система фильтров
 
* Страница Tasks отображает все задачи воркспейса с поддержкой многоуровневой фильтрации.
* Компонент `TaskFilters` принимает фильтры через `v-model` и позволяет фильтровать задачи по:
  * **Статусу** — мультивыбор через чекбоксы.
  * **Приоритету** — мультивыбор через чекбоксы.
  * **Исполнителю** — мультивыбор участников воркспейса по имени.
  * **Тегам** — выбор через теги-кнопки с подсветкой активных.
  * **Дедлайну** — быстрые пресеты: Overdue, Today, This week.
  * **Переключатели:** показывать только повторяющиеся задачи или только задачи с подзадачами.
* Состояние всех фильтров синхронизируется с URL (`query string`) — ссылку можно скопировать, поделиться или обновить страницу без потери фильтров.
* Десериализация фильтров из URL при монтировании: строки, булевы значения, массивы.
* Доступные значения фильтров (теги, статусы, приоритеты, исполнители) вычисляются динамически из текущих задач воркспейса.
 
### Фильтрация и группировка
 
* Задачи на сегодня (`todayTasks`) — задачи с дедлайном на текущую дату, исключая архивные.
* Активные задачи (`activeTasks`) — все задачи, кроме архивных.
* Выполненные задачи (`completedTasks`).
* Архивные задачи (`archivedTasks`).
 
### Подзадачи
 
* Создание и отображение подзадач у задачи.
* Переключение статуса выполнения подзадачи с оптимистичным обновлением UI.
 
### Повторяющиеся задачи
 
* При загрузке задач автоматически проверяется и создаются копии повторяющихся задач за текущую дату (`RecurringTaskService`).
* Хранение `original_task_id` для связи копии с шаблоном.
 
### Архив
 
* Перемещение задачи в архив и восстановление из архива с оптимистичным обновлением.
 
### Завершение задачи и XP
 
* Переключение статуса задачи между `done` и `backlog` (`toggleTaskCompletion`) с получением данных о заработанном XP и возможном повышении уровня.
* После изменения автоматически обновляется прогресс пользователя.
 
## Таймер
 
* **Запуск таймера** для конкретной задачи — задача автоматически переводится в статус `in_progress`.
* **Отслеживание времени** в реальном времени через `requestAnimationFrame` с обновлением раз в секунду.
* Форматированное отображение прошедшего времени: `мм:сс` или `ч:мм:сс`.
* **Остановка таймера** — сохраняет сессию в историю, вычисляет потраченные минуты (минимум 1 минута).
* **Персистентность:** активный таймер и последняя сессия сохраняются в `localStorage` — таймер продолжает работать при перезагрузке страницы.
* **История таймера:** хранение записей о каждой рабочей сессии с привязкой к задаче и шаблону повторяющейся задачи.
* Статистика по таймеру: суммарное время за сегодня, история за день.
* Удаление записей из истории.
 
## XP и система прогрессии
 
* Пользователь зарабатывает XP за выполнение задач.
* Система уровней с таблицей требований (`LevelRequirement`) — каждый уровень требует определённого суммарного XP.
* Отображение прогресса к следующему уровню в процентах.
* Ежедневные достижения: задачи выполнено сегодня, XP заработано сегодня.
* **Серия (streak):** подсчёт непрерывных дней активности на основе `DailyCompletion`.
* Retry-механизм при загрузке прогресса — до 5 попыток с задержкой.
 
## Аналитика (Analytics)
 
* Страница Analytics предоставляет визуальную статистику по задачам воркспейса на основе графиков ApexCharts.
* **Completing tasks** — линейный график выполненных vs запланированных задач по неделям.
* **On-Time vs Overdue** — столбчатая диаграмма соотношения задач, выполненных в срок и просроченных, по неделям.
* **Team Performance** — горизонтальная гистограмма выполненных задач по каждому участнику команды.
 
## Календарь (Calendar)
 
* Страница Calendar отображает задачи воркспейса на недельной сетке с временными слотами (09:00 — 22:00+).
* Интеграция с **FullCalendar** (timegrid-представление): задачи отображаются в виде блоков по времени начала и оценочной продолжительности.
* Визуальное различие задач по типу: повторяющиеся задачи выделены отдельным цветом, обычные — другим.
* Каждый блок показывает: название задачи, время начала и окончания, продолжительность, метку повторяющейся задачи.
* Подсветка текущего дня на сетке.
* Поддержка часового пояса пользователя через `moment-timezone`.
 
## Уведомления
 
* Загрузка списка уведомлений пользователя.
* Отметка уведомлений как прочитанных.
* Удаление уведомлений.
* Счётчик непрочитанных уведомлений.
* Специальная выборка уведомлений типа `workspace_invitation` — приглашения в воркспейс отображаются отдельно.
 
## Тосты
 
* Глобальная система toast-уведомлений: тип `success` / `error`.
* Автоматическое скрытие через 3 секунды.
* Несколько тостов могут отображаться одновременно.
 
## Хранение данных и синхронизация
 
* Серверное хранение всех критичных данных (профили, задачи, воркспейсы, история таймера, прогресс) в Supabase с клиентским реактивным кэшем.
* Восстановление сессии при инициализации приложения и отслеживание изменений состояния авторизации через `onAuthStateChange`.
* Кэширование профилей по `userId` с возможностью точечной и полной очистки.
 
## Архитектура и производительность
 
* Модульная структура сторов: каждый домен (auth, profile, tasks, workspaces, timer, progress, notifications) — отдельный Pinia store.
* Оптимистичные обновления UI с автоматическим откатом при ошибке для всех критичных операций (статус задачи, архив, завершение).
* Retry-логика для нестабильных операций (загрузка профиля, прогресса, создание профиля после регистрации) — до 5 попыток с экспоненциальной задержкой.
* Отдельные сервисные слои (`TaskService`, `WorkspaceService`, `TimerService`, `ProgressService` и др.) — сторы не делают запросы напрямую.
* Защита readonly-экспортов реактивных рефов из сторов.
 
## UI / UX
 
* Минималистичный, современный дизайн с кастомными компонентами.
* Мгновенная визуальная реакция на действия: оптимистичные обновления, спиннеры загрузки, toast-уведомления об успехе и ошибках.
* Полная адаптивность интерфейса для мобильных и десктоп-устройств.
 
## Страницы и маршрутизация
 
Приложение использует **Vue Router** с hash-history (`createWebHashHistory`) и навигационными guard-ами:
 
* **`/`** — Welcome (только для гостей).
* **`/signup`**, **`/signin`** — Регистрация и вход (только для гостей; авторизованные перенаправляются на дашборд или онбординг).
* **`/onboarding`** — Онбординг нового пользователя (требует авторизации; пропускается, если воркспейс уже создан).
* **`/forgot-password`**, **`/reset-password`** — Восстановление и сброс пароля.
* **`/invite/:token`** — Принятие приглашения в воркспейс по токену.
* **`/dashboard`** — Основной лейаут дашборда (требует авторизации), содержит дочерние маршруты:
  * **`today`** — задачи на сегодня.
  * **`tasks`** — полный список задач с фильтрами.
  * **`kanban`** — Kanban-доска.
  * **`calendar`** — календарный вид задач.
  * **`analytics`** — аналитика и графики.
  * **`team`** — управление командой воркспейса.
 
Навигационный guard автоматически устанавливает `document.title` по метаданным маршрута и перенаправляет пользователя в зависимости от состояния авторизации и онбординга.

## Технологии
 
* **Vue 3 (Composition API)**
* **TypeScript**
* **Pinia** — управление состоянием
* **Vue Router** — маршрутизация с hash-history и навигационными guard-ами
* **Supabase** — база данных, аутентификация, хранилище файлов (Storage)
* **FullCalendar** (`@fullcalendar/vue3`, `timegrid`, `interaction`) — календарный вид задач
* **ApexCharts** (`apexcharts` + `vue3-apexcharts`) — графики аналитики
* **vue-draggable-plus** — drag-and-drop для Kanban-доски
* **moment-timezone** — работа с часовыми поясами
* **Vee-Validate + Yup** — валидация форм
* **Vite** — сборка и dev-сервер
* **Vitest** — юнит-тестирование

## Установка
 
```bash
git clone https://github.com/Fizz1eOut/FlowTrack.git
cd FlowTrack
```
 
### Frontend
 
```bash
cd frontend
npm install
npm run dev
```
 
### Backend
 
```bash
cd backend
npm install
npm run dev
```
 
> Для работы приложения необходимо создать файл `.env` в папке `frontend` и добавить переменные окружения Supabase:
> ```
> VITE_SUPABASE_URL=your_supabase_url
> VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
> ```
